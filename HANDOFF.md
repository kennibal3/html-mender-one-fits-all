## 11. 当前格：弹窗内部图片选择与替换

### 给新手的解释

现在老师从左侧进入弹窗后，已经可以点击并修改真实文字。下一步需要证明弹窗里的图片也能像首页图片一样被选中和替换。

本格只打通“进入弹窗 → 选中图片 → 替换图片 → 返回并重新进入仍保留”这一条最小闭环，不提前做布局编辑、深层 A1–A5 或同源子页面。

### 严格执行顺序

1. 先阅读现有图片扫描、选中、替换、撤销和资源序列化逻辑，不另建一套深层图片编辑器。
2. 先写浏览器端失败测试：从场景树进入含图片弹窗，点击可见图片并通过现有替换入口换成测试图片。
3. 在同一测试中返回首页再重新进入，证明图片修改仍在真实节点上；原父节点、顺序、表单和原事件不得受损。
4. 再写满足测试的最小实现；若现有能力已经完整可用，则只补证据。
5. 回归弹窗文字编辑、真实点击位置同步、场景树强制进入、嵌套弹窗、干净序列化和 A1–A5 平面模型。
6. 运行全量测试、相关 Chromium 测试、语法检查、`git diff --check` 和高危依赖审计，逐项留证据。

### 已确认设计与决策记录（2026-07-22）

本格只覆盖弹窗中的普通 `<img>`。老师从左侧“课件画面”进入弹窗后，点击真实图片并复用现有“替换图片”入口；替换写回同一个活动节点，返回首页再重新进入时仍保留。原父节点、顺序、表单值、原点击事件和隐藏状态不得受损。

采用“证据优先、最小补丁”方案：先在脱敏弹窗测试副本中加入一张普通图片，再写浏览器失败测试。测试使用小型本地 PNG，不访问网络、不修改真实课件；完整覆盖图片扫描与选中、真实文件选择、替换、撤销、重做、返回与重新进入、干净 HTML 序列化、节点身份和 console 零报错。

如果现有图片选择框已经完整可用，只保留测试证据，不修改生产代码。如果测试证明弹窗内图片无法选中，只在现有场景点击处理器中增加普通图片选择的最小分支，继续复用当前图片扫描、工具条、文件输入、Data URL、撤销和序列化逻辑。若修改编辑器运行时，必须同步升级运行时迁移版本，确保旧任务重新注入修复。

本格不是互动行为变更，因此不修改导出互动运行时 `html-slide-mender-interactions.js`。已明确排除 `<picture>`、CSS 背景图、独立深层图片编辑器、布局编辑、深层 A1–A5、同源子页面和新资源管理系统。

备选方案已拒绝：不预先扩展所有场景点击行为，避免修改本来已经工作的路径；不为弹窗建立第二套图片编辑器，避免重复维护撤销、保存和导出。

### 本格不做

- 弹窗内部布局编辑与深层 A1–A5 配置。
- 同源子 HTML 页面。
- 人工登记、合并误识别和高级诊断。

### 实现结果

- 失败测试先证明：从场景树进入弹窗后，直接点击普通图片无法选中。
- 最小实现让场景中的普通 `<img>` 复用现有图片选择、替换、撤销、重做和保存能力。
- 图片仍是原来的真实节点，不新增包裹层，也不改变原父节点。
- 图片工具条在弹窗内保持可点击，并避开已打开的页面侧栏。
- 返回首页时清理临时层级样式，重新进入仍能看到替换结果。
- 运行时构建标识升级为 `2026-07-22-deep-modal-image-frame-v3`，服务端注入版本升级为 31，旧任务会重新注入本格修复。
- 用户测试暴露的尺寸问题根因是：场景图片本身充当固定图片框时，又被错误地设为父容器的 `100%` 宽高。修复后该路径只锁定替换前的像素宽高，新图片以 `cover` 方式放入原图片框，不改变真实节点和父节点。
- 直接点击课件按钮的路径还暴露出另一层根因：包含标题和控件的整块内容区被误识别成图片框。当前实现只把 `<picture>` 或仅含图片资源的父元素视为图片框，普通内容区不再被写入图片填充样式。
- 顶部工具栏现在插入仅编辑态生效的安全留白；退出编辑或干净序列化时不会把留白写入课件成品。
