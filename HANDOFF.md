## 5. 本轮已确认的产品边界

复杂 HTML 采用以下兼容边界：

> 本地及同源内容尽量完整编辑；跨域、封闭或浏览器禁止访问的内容只做外层管理和预览。

已确认的关键决策：

- 导航：场景树 + 直接进入 + 面包屑。
- 发现：静态扫描 + 运行发现 + 人工补充。
- 数据：原始文件 + 场景清单 + 编辑补丁。
- 状态：项目—页面—场景—测试四层模型。
- 脚本：观察—编辑—测试—兼容—接管分层运行。
- 弹窗：原生优先 + 活动节点 + 静态副本兜底。
- 保存：自动草稿 + 手动正式版本。
- 导出：工作副本 + 场景清单 + 独立运行包。
- 安全：默认隔离预览 + 可选可信项目模式。
- 诊断：教师模式默认隐藏，高级维护模式开启。
- 性能 A 档：50 个 HTML、200 个场景、500MB。

B1–B8 全部纳入正式范围：

- B1 答题判定与即时反馈。

### 6.1 场景模型

新增 `src/scene-model.js`：

- 场景清单版本：`1.0`。
- 生成稳定页面场景 ID。
- 从互动清单生成弹窗场景。

- 无效互动清单不会导致页面场景丢失。

`src/server.js` 已接入场景清单：

- HTML/ZIP 导入时生成。
- 页面保存时更新。
- 新增、复制、排序、删除和恢复页面时更新。
- 服务重开时重新检查并持久化。
- `publicProject` 返回场景清单。

### 6.2 活动节点弹窗

导出运行时和编辑预览不再复制弹窗目标：

- 使用占位节点保存原位置。
- 移动真实活动节点进入弹窗。
- 保留原 JavaScript 事件、表单值、媒体状态和内部互动标识。
- 关闭后恢复原父节点、原顺序、样式、隐藏状态和焦点。
- 使用弹窗栈支持多层打开和后进先出关闭。
- 关闭子弹窗后恢复父弹窗内容与状态。

### 6.3 场景事件

编辑预览和导出运行都会派发：

```text
hsm-scene-event
```

事件包含：

- `scene.entered` / `scene.exited`
- `sceneId`
- `interactionId`
- `depth`
- `preview`
- 时间戳与模式版本

这些事件供后续场景树、四层状态和路径重放使用。
