# AGENTS.md — HTML Mender

面向教师的**本地** HTML 课件编辑器:导入 HTML/ZIP → 多页管理 → 元素/互动编辑 → 导出标准 HTML/ZIP →(可选)Electron Windows 打包。当前为单机 + 本地文件存储。

本文件是常驻硬规矩,每次会话都要遵守。**当前任务的具体验收清单在 `docs/GOAL-互动功能完整验收.md`,由 `/goal` 指向,不在本文件展开。**

## ★ 最重要的架构约束(违反即算未完成)

互动有**两条运行时代码路径,必须同时修改、行为一致、各有测试**:
- `vendor/html-slide-mender/assets/html-slide-mender-runtime.js` — 编辑时的可视化预览。
- `vendor/html-slide-mender/assets/html-slide-mender-interactions.js` — 导出课件后在浏览器里实际运行互动的逻辑。
只改一条、漏另一条,一律视为未完成。历史上"导出后失灵"几乎全来自漏改 interactions.js。

## 命令
- 启动:`npm start`(访问 http://127.0.0.1:8787/)
- 测试:`npm test`
- 完成前必须 npm test 全绿、且导出文件的端到端测试全绿。若命令与仓库不符,以 package.json 的 scripts 为准并先纠正本行。

## 硬规矩
- 深度优先,禁止广度优先:一次只做一种互动,每格跑通留证据才开下一种。
- 每格必须有通过证据(自动化测试或可复现端到端脚本),不许口头声称"应该没问题"。
- 两条运行时路径都要改到。
- 导出后必须可双击独立运行,不依赖本地服务,console 零报错。
- 不许越界:不改无关代码,不加未要求的功能。
- 不许破坏用户数据:保存/导出/版本恢复都不得损坏已有课件。
- 版本恢复要连互动一起回滚。
- 规格不清就停下来问,不要自行假设。

## HTML 独有互动(B 类)额外要求
除"能跑"外必须多验逻辑正确性:判定/分支/命中在所有输入下都正确,覆盖正例、反例、边界、非法输入,杜绝"点了有反应但答案永远判对"。B 类在配置界面、导出 HTML 的 data 属性、代码注释中都要有可识别标记(如 data-hm-interaction="quiz")。

## 完成的定义
以 /goal 当前目标的"完成条件"为准。合并前由人工审查 diff,自动化不替代人工审查。
