# HTML Mender Windows 桌面版设计

## 已确认需求

- 提供 Windows 10/11 64 位安装版和免安装版。
- 同事无需单独安装 Node.js，双击应用即可使用。
- 保留现有 HTML、ZIP、多页面编辑、逐页保存、版本记录和完整导出功能。
- 默认在用户桌面创建 `HTML Mender 工作区`，集中保存项目和历史版本。
- 移动或删除免安装 EXE 不影响已经保存的工作区数据。
- 继续保留网页版启动方式，未来可以部署到服务器。
- 第一版不包含账号、云同步和自动更新。

## 架构

Electron 主进程启动现有 Express 服务，等待服务监听随机本地端口后再打开桌面窗口。应用关闭时同步关闭服务，避免残留后台进程和固定端口冲突。

服务的数据根目录通过 `HTML_MENDER_DATA_DIR` 注入。桌面版默认使用 `桌面/HTML Mender 工作区`；普通网页版未设置该变量时继续使用项目内的 `data` 目录，保持向后兼容。

## 打包产物

- `HTML-Mender-Setup.exe`：标准安装程序，创建开始菜单和桌面快捷方式。
- `HTML-Mender-Portable.exe`：免安装程序，双击直接运行。

Windows 安装包由 GitHub Actions 的 Windows 执行环境生成，避免依赖 Mac 上不稳定的跨平台打包环境。

## 数据与安全

- 服务只监听 `127.0.0.1`，不暴露到局域网。
- 工作区包含上传缓存、编辑输出、项目版本和导出压缩包。
- 窗口只允许打开本地应用地址；外部链接交给系统默认浏览器。
- 打包时不携带开发电脑上的 `data`、日志、测试文件和已有用户项目。

## 决策记录

- 选择 Electron：可以最大程度复用现有 Node/Express 与网页界面。
- 同时提供安装版和免安装版：分别覆盖长期使用和快速分发。
- 工作区放在桌面独立文件夹：用户容易找到、整理和备份。
- Windows 原生环境构建：比在 Mac 上跨平台生成 EXE 更可靠。
