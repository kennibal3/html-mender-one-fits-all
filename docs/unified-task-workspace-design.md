# 统一任务工作台设计

## 已确认目标

- 单个 HTML、多个 HTML 和 ZIP 都创建为由用户命名的本地任务。
- 主界面显示最近保存任务，应用关闭重启后可恢复任务、页面和版本记录。
- 每个页面独立保存版本，并可预览、下载和恢复旧版本。
- 恢复旧版本时生成一个新版本，不删除后续历史。
- 编辑页面可以返回任务、切换前后页；存在未保存修改时必须提醒。
- 保存成功显示当前页版本号，并同步页面列表和最近任务时间。
- 可以把当前页面的干净完整 HTML 复制到剪贴板，不包含编辑器代码。
- 每次完整导出都由用户通过系统“另存为”窗口选择目录和文件名。

## 统一模型

现有 ZIP 项目模型升级为任务模型。HTML 单页、HTML 多页和 ZIP 只在创建方式与导出格式上不同，共用任务索引、页面导航、版本、恢复和导出逻辑。

任务保存在 `HTML Mender 工作区/projects/<任务ID>`，`meta.json` 记录任务名称、类型、页面顺序、逐页版本、创建时间和最近修改时间。元数据采用临时文件后原子替换，降低中断写入造成的损坏。

## 版本规则

每页独立编号 `v001`、`v002`。版本文件只保存该页的干净 HTML，项目资源继续由任务的 `output` 目录提供，避免每次保存都复制整个大体积 ZIP。恢复历史版本会把历史 HTML 写回当前页面并创建该页的新版本。

## 编辑规则

固定任务工具栏提供保存版本、复制 HTML、返回任务、上一页和下一页。离开前比较当前干净 HTML 与最近保存内容；未保存时提供保存后离开、放弃修改和继续编辑。编辑器内置退出按钮采用相同规则。

## 导出规则

单页 HTML 任务完整导出为 HTML；多页 HTML 和 ZIP 任务完整导出为 ZIP。Electron 通过安全 preload 与 IPC 打开原生保存窗口，网页版退回浏览器下载。

## 决策记录

- 选择统一任务模型，而不是维护独立的 HTML 批次系统。
- 选择逐页轻量版本，而不是为每次页面保存复制完整项目资源。
- 旧版本恢复采用追加新版本，保证历史不可逆删除。
- 第一版保持单机、本地存储，不增加账号、云同步或多人协作。
- 最近任务默认显示 10 个，全部任务按最近修改时间排序。
