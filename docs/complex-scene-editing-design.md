# HTML Mender 复杂场景编辑设计

> **当前执行边界（2026-07-19）：** 本文件是长期架构来源文档。深层内容编辑第一版只执行 `docs/GOAL-深层内容编辑第一版.md` 的最小子集；两者冲突时以当前 GOAL 为准，不得把 iframe、SPA、多学习者或 B1–B8 提前拉入。

## 1. 文档目的

HTML 课件可能同时包含多个 HTML 页面、路由、弹窗、iframe、标签页、动态生成内容和多层互动。当前按“页面 + 顶层元素”编辑的模型无法完整覆盖这些情况，典型表现是首页互动正常，但弹窗或跳转后的内部互动失效。

本设计将编辑对象从平面页面升级为层级化的“课件场景”，使教师能够进入、编辑、测试、保存和导出本地及同源范围内的深层内容，同时保留原课件已有交互。

本文记录已经确认的产品边界和架构决策。后续代码改动必须分阶段实施，每个阶段单独确认、测试和验收。

## 2. 目标用户与设计原则

目标用户是普通教师。教师不需要理解 HTML、路由、iframe、DOM、事件监听或前端框架。

设计原则：

- 看到哪里，编辑哪里；隐藏场景也能通过结构导航进入。
- 用户拥有并导入的本地、同源内容尽量完整编辑。
- 优先保留原网页结构、脚本、事件和运行状态。
- 无法安全深入的内容只管理外层，并说明限制原因。
- 编辑器预览、正式导出运行和未来学生数据互相隔离。
- 所有导出产物仍是可独立运行的标准 HTML 或完整 ZIP。
- 复杂功能默认不干扰教师，高级诊断按需开启。

## 3. 正式兼容边界

采用“本地及同源内容完整支持，跨域和封闭内容只做外层管理与预览”的边界。

### 3.1 完整支持范围

- 多个本地 HTML 之间的连续跳转。
- 弹窗内部互动和多层弹窗。
- 同源 iframe 及同源 iframe 的继续嵌套。
- Hash、History API 和常见 SPA 路由形成的视图。
- 点击后由 JavaScript 动态生成的 HTML 元素。
- 标签页、折叠面板、轮播图等单文件多视图结构。
- 表单、答题、得分、显示隐藏和条件分支状态。
- 视频时间点触发的显示、提问或跳转互动。
- SVG 内可识别的图形元素。
- 新窗口、下载和特殊链接的安全配置。
- 场景进入、退出、返回和重新打开后的状态恢复。
- HTML、ZIP 导出的路径、资源和运行一致性。

### 3.2 有限支持范围

- React、Vue 等本地项目：管理浏览器中实际呈现的场景，不承诺修改框架源代码；直接 DOM 修改可能被框架重新渲染覆盖。
- 开放 Shadow DOM：允许专门适配后进入，封闭 Shadow DOM 不进入内部。
- Canvas、小游戏和复杂图表：可以编辑外层容器和整体互动，不直接编辑画布内部对象或游戏逻辑。
- 原网页 JavaScript：提供事件隔离、冲突发现和兼容模式，但不保证理解所有第三方脚本。

### 3.3 只预览或外层管理

- 跨域 iframe 和第三方在线网页。
- 封闭 Shadow DOM。
- 加密或高度混淆的脚本逻辑。
- DRM 音视频的内部行为。

如果用户拥有第三方页面源码并将其完整导入项目，可重新按本地或同源内容处理。

### 3.4 B1–B8 正式互动范围

B1–B8 全部纳入正式建设范围，并与复杂场景架构交错实施：

- B1 答题判定与即时反馈：单选、多选、填空、判断、解析及正反例判定。
- B2 条件分支：按照答案、选择或变量进入不同页面、弹窗、iframe 或动态场景。
- B3 拖拽交互：配对、分类、排序和拖入空格，支持鼠标与触摸。
- B4 输入即时响应：通过可视化规则配置变量、公式和页面响应，不开放任意 JavaScript。
- B5 状态与计分：支持题目分值、部分得分、尝试次数扣分、总分、进度和达标解锁。
- B6 随机化：支持抽题、题序、选项和拖拽项目随机化；同一学习者同一次练习保持稳定，重新练习后才重新生成。
- B7 计时与倒计时：每个计时器可以分别配置离开场景后暂停或继续，并支持超时动作和重开恢复。
- B8 媒体精确联动：使用媒体时间轴配置时间点或时间段互动，并提供教师常用的轻量媒体编辑。

B8 的轻量媒体编辑包括开头结尾裁剪、分段、排序、音量、淡入淡出、封面、字幕和简单画面裁切。不包含专业多轨、绿幕、复杂特效和专业关键帧；此类内容继续使用专业工具制作后导入。

## 4. 场景模型

编辑对象由“项目中的平面页面”升级为以下层级：

```text
项目
└── 页面或路由
    └── 弹窗、iframe、标签页或动态视图
        └── 内部互动或更深层场景
```

每个场景至少记录：

- 稳定场景 ID、名称和类型。
- 所属 HTML 文件或路由。
- 父场景和子场景关系。
- 进入场景需要执行的动作或满足的条件。
- 对应元素、选择器、节点标识或 iframe 地址。
- 场景编辑补丁和互动配置。
- 状态作用域及进入、退出规则。
- 识别来源、可信度和兼容说明。

场景不是 HTML 文件的副本。多个弹窗、标签页和动态视图可以共同引用同一个真实 HTML 文件。

## 5. 场景导航

采用“场景树 + 直接进入”的混合导航方式。

- 教师可以像使用真实课件一样点击并进入深层场景。
- 选中弹窗、iframe 或标签页时，提供“进入内部编辑”。
- 左侧场景树展示已经发现并确认的场景。
- 顶部使用面包屑显示当前位置，例如“首页 > 课程介绍弹窗 > 练习题 > 答错反馈”。
- 原入口损坏或无法点击时，仍能从场景树强制进入。
- 返回父场景时恢复滚动位置、弹窗层级、焦点和测试状态。
- 新发现的动态场景先标记为“待确认”，不直接污染正式结构。

## 6. 场景发现

采用“静态扫描 + 运行发现 + 人工补充”的混合方式。

### 6.1 静态扫描

- 扫描本地 HTML、同源链接、iframe 和常见路由。
- 识别隐藏弹窗、标签页、折叠面板和候选视图。
- 建立初始场景树，并记录识别可信度。

### 6.2 运行发现

- 监测地址、DOM、弹窗、iframe 和可见区域变化。
- 记录新场景的触发器、进入路径和必要条件。
- 对内容相同、路径等价的重复场景进行合并建议。
- 检测循环路由，避免无限扫描。

### 6.3 人工补充

- 教师可以框选区域并选择“作为独立场景编辑”。
- 可以合并误识别场景，或标记为普通动画变化。
- 无法进入的跨域内容标记为外部场景。

## 7. 保存与版本模型

采用“原始文件 + 场景清单 + 编辑补丁”。

- 一个真实 HTML 文件只保存一份。
- 场景清单记录层级、进入路径、状态和互动关系。
- 编辑补丁准确作用于对应文件和元素。
- iframe 指向另一个本地 HTML 时，修改对应子文件。
- 保留原始导入文件作为可恢复基线。

保存机制采用“自动草稿 + 手动正式版本”：

- 编辑实时写入本地增量草稿，场景切换不丢失修改。
- 图片、音视频等大资源在历史中去重存储。
- 应用异常退出后可以恢复最近草稿。
- 用户点击“保存版本”时才生成可命名、可恢复的正式项目版本。
- 恢复历史版本前自动建立当前状态临时恢复点。
- 导出时如存在未确认草稿，允许选择当前修改或上次正式版本。
- 互动测试产生的临时显示状态不写入草稿。
- 学生练习数据按课件和本地学习者分别保存，不混入课件编辑版本。
- 学习数据通过统一存储接口写入当前浏览器数据库，为未来切换云端存储适配器保留边界。

## 8. 四层状态模型

采用“项目—页面—场景—测试”四层状态：

- 项目状态：全课件共享，例如姓名、总分和章节完成情况。
- 页面状态：仅属于某个 HTML 或路由，例如当前题号和展开区域。
- 场景状态：属于某个弹窗、标签页或 iframe，例如局部选项。
- 测试状态：仅用于编辑器临时预览，退出测试后不进入正式课件。

默认规则：

- 子场景继承父场景允许共享的状态。
- 关闭弹窗后恢复父场景滚动位置和互动状态。
- 返回场景可配置为保持上次状态或重新进入时重置。
- 同名元素不自动共享状态，必须显式绑定变量。
- iframe 通过受控消息通道共享允许的状态。
- 历史版本恢复结构和配置，不恢复某次临时测试状态。

教师界面只展示“每次打开重置”“本页内保持”“整个课件保持”“关闭后清除”等易理解选项。

本地阶段支持同一台电脑上的多个学习者档案：

- 档案使用姓名或编号区分，进度、答案、分数、计时和互动状态彼此隔离。
- 每个课件可以设置课堂演示模式或学生练习模式。
- 演示模式不要求身份，也不写入个人成绩。
- 练习模式允许选择、新建、切换、重新开始和删除本地档案，并记住上次选择。
- 数据模型预留课件 ID、学习者 ID、场次 ID、互动 ID 和数据版本，未来可以映射到云端学生账户。

## 9. 原脚本兼容与运行分层

采用“观察—编辑—测试—兼容—接管”的分层运行方式。

- 观察模式：记录原网页行为，不阻止原事件。
- 编辑模式：使用编辑层选择和修改元素，避免误触发原互动。
- 互动测试模式：共同运行原课件事件和 HTML Mender 互动，并记录执行顺序。
- 兼容模式：指定区域继续完全使用原网页逻辑。
- 接管模式：只有教师明确选择后，才以 HTML Mender 互动替换原动作。

系统不随意删除原脚本、内联事件或事件属性。发生冲突时，允许选择保留原动作、使用新动作或按顺序执行。无法安全编辑的组件只降级局部区域，不影响整个课件。

## 10. 活动节点弹窗

采用“原生弹窗优先 + 活动节点弹窗 + 静态副本兜底”。

- 原课件已有弹窗机制时，优先触发原弹窗。
- HTML Mender 配置的弹窗默认显示真实活动节点，不复制内部内容。
- 关闭后恢复原位置、原样式、焦点和状态。
- 多层弹窗分别保存返回位置和焦点。
- 只有用户明确选择静态内容副本时才克隆内容，并提示副本内部互动可能不继承。
- 活动节点不能安全弹出时，局部降级到兼容模式。

该决策用于避免克隆造成事件监听、节点标识、表单、播放器和第三方组件状态丢失。

## 11. 导出模型

采用“工作副本 + 场景清单 + 独立运行包”。

- 编辑过程中在任务工作副本中应用修改。
- 项目版本保存文件、场景清单、互动和状态规则的一致性快照。
- 导出时将确认后的修改写入导出副本。
- 本地互动运行逻辑和场景运行清单随 ZIP 一起导出。
- 导出课件不依赖 HTML Mender 或云端服务即可运行。
- 导出页面不包含编辑器工具条和维护界面。
- 原课件自身依赖在线服务时，在导出报告中说明。
- 导出课件保留课堂演示模式和学生练习模式配置；练习数据仍只写入当前设备，不随 ZIP 携带其他学习者的历史记录。

导出前检查页面、路由、iframe、互动目标、循环跳转、不可达场景、资源路径、文件名大小写、本地文件安全限制和运行一致性。

## 12. 安全模型

采用“默认隔离预览 + 可选可信项目模式”。

默认隔离状态下：

- 工作台和课件运行在权限隔离环境中。
- 课件脚本不能直接调用任务管理、删除、导出等本地接口。
- 编辑器通过受控消息通道读取和修改内容。
- 同一个课件包内的 HTML 和同源 iframe 可以按规则通信。
- 新窗口、下载、文件访问和网络请求按权限控制。
- 脚本错误保存在本地，不上传用户课件和数据。

可信项目模式只对当前任务生效。开启前说明将放开的权限，随时可以恢复隔离模式，高风险动作仍需明确确认。

## 13. 诊断与维护模式

系统提供互动健康检查、路径重放和技术详情，但默认对教师隐藏。

- 默认教师模式不显示诊断面板和技术术语。
- 普通互动正常时不出现任何诊断入口干扰。
- 失败时只显示简短、可理解的提示。
- 在任务设置中开启“高级维护模式”后，才显示项目诊断、路径重放和技术详情。
- 高级维护模式状态不写入导出课件。
- 导出前检查在后台执行；只有阻断性问题才打扰教师。

健康检查覆盖触发元素、目标元素、遮挡和锁定、事件阻止、活动节点与静态副本、iframe 权限、路由出现、脚本异常、返回状态及导出结果。

## 14. 性能与容量

正式保障 A 档规模：

- 不超过 50 个 HTML。
- 不超过 200 个场景。
- 项目资源不超过 500MB。

实现要求：

- 导入时优先扫描结构和引用，不立即解码全部大资源。
- 场景树按需加载，缩略图后台分批生成。
- 动态监测只作用于当前活动页面和已打开场景。
- 相同大资源只存储一份。
- 达到保障容量 80% 时提示拆分，但不直接阻止编辑。
- 超出保障范围进入大型项目模式，降低缩略图和全量扫描频率。
- 场景超过 10 层时提示结构复杂，但不设置很小的硬限制。
- 单个场景异常只隔离该场景，不使整个任务退出。

## 15. 验收标准

每个支持的场景类型至少验证以下闭环：

1. 导入后能够发现或人工登记。
2. 可以从场景树和真实点击两种路径进入。
3. 可以在深层场景中选中并编辑元素。
4. 可以配置、预览和返回互动。
5. 返回父场景后位置和状态正确。
6. 自动草稿和正式版本均可恢复。
7. 关闭应用后重新打开关系不丢失。
8. HTML 或 ZIP 导出后无需编辑器即可运行。
9. 原脚本未被无提示删除或覆盖。
10. 无法深入的内容有明确、可理解的限制说明。

典型强制回归路径：

- 首页 → 弹窗 → 弹窗内按钮 → 显示隐藏 → 返回首页。
- 首页 → 子 HTML → 二级弹窗 → 返回上一页。
- 首页 → 同源 iframe → iframe 内互动 → 返回父页面。
- SPA 路由 → 动态标签页 → 条件分支 → 历史恢复。
- 保存草稿 → 保存版本 → 关闭重开 → ZIP 导出 → 独立运行。
- 学习者 A 作答与计分 → 切换学习者 B → 确认两者进度隔离 → 重开后分别恢复。
- 随机抽题 → 刷新和重开后顺序保持 → 重新练习后生成新顺序。
- 倒计时离开暂停/继续两种规则 → 超时分支 → 重开恢复。
- 媒体时间点弹题 → 作答后继续播放 → 独立导出运行。

## 16. 分阶段实施顺序

后续采用“建立一个必要架构支点，立即完成一个 B 类互动闭环”的交错顺序。各阶段分别确认，不同时铺开多个未完成互动：

1. 建立最小场景模型并升级活动节点弹窗，打通“首页 → 弹窗 → 内部互动 → 返回”。
2. 建立 B5 状态基础设施，包括四层变量接口、本地存储适配器、多个学习者档案及演示/练习模式；该阶段只建设后续互动必需的公共基础。
3. 完成 B1，并结合 B1 补全 B5 的计分、进度和解锁闭环。
4. 扩展多 HTML、同源 iframe 和场景树目标导航，随后完成依赖 B1 结果的 B2。
5. 实现动态 DOM、SPA、标签页和人工补充场景，随后完成只使用可视化公式的 B4。
6. 建立适配页面缩放、弹窗和 iframe 的统一坐标层，随后完成 B3。
7. 建立按学习者和练习场次保存的稳定随机实例，随后完成 B6。
8. 建立场景生命周期调度器，随后完成可配置暂停或继续规则的 B7。
9. 建立媒体时间轴，完成 B8 和已确认范围内的轻量媒体编辑。
10. 完成自动草稿、正式版本、独立 ZIP、隔离预览、可信模式、隐藏维护诊断、A 档性能和 Windows 回归等统一加固。

每个代码阶段都应先补充或更新自动化测试，再实现功能，最后执行相关测试、完整测试和安全依赖检查。当前已经稳定的页面管理与 A1–A5 互动能力必须持续通过回归测试。
