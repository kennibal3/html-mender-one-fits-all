# HTML 教学互动课件能力与平台协议设计

## 文档状态

- 状态：已确认设计方向，等待分阶段实施。
- 产品：HTML Mender。
- 当前阶段：本地单机编辑、预览、版本管理与导出。
- 未来阶段：课件上传教学平台，由平台提供学生账号、课程上下文和学习记录。
- 核心原则：教学设计永久保存，测试过程临时保存，学生学习结果由平台正式保存。

## 已确认目标

- 把 HTML Mender 升级为老师可以像修改 PPT 一样操作的互动课件编辑器。
- 老师通过可视化界面创建互动，不需要理解或编写 HTML、CSS、JavaScript。
- 支持单个 HTML/HTM、多个 HTML 和包含完整资源的 ZIP 项目。
- 课件在本地离线状态下可以编辑、预览和独立运行。
- 课件将来上传平台后，可以直接接入学生账号和学习记录，不需要重新制作。
- 互动覆盖内容讲解、媒体、练习、虚拟实验、课堂游戏、教师工具和无障碍使用。
- 交互功能应服务教学目标，不以动画数量或视觉效果作为产品目标。

## 产品定位

HTML Mender 是面向老师的可视化 HTML 互动课件生成、编辑、版本管理和发布工具。

老师看到的是接近 PPT 的编辑界面；系统内部保存的是 HTML、CSS、JavaScript、图片、音频、视频和结构化互动配置。HTML 课件项目相当于本产品的原生课件格式。

主要输出形式：

- 单个 HTML/HTM：适合单页或资源已内嵌的课件。
- 完整 ZIP：包含多页面、样式、脚本、字体和媒体资源。
- 平台课件包：在 ZIP 基础上包含互动清单和协议版本。
- Windows 应用：通过桌面外壳运行 HTML 课件。
- PDF 或图片：仅用于静态发布，互动能力不保留。

## 当前边界

当前需要实现：

- 老师在本地创建、配置和预览教学互动。
- 互动规则与课件版本一起保存和导出。
- 本地预览使用匿名临时会话，不上传学习数据。
- 为未来平台记录预留稳定的互动编号和标准事件。

当前明确不实现：

- 学生账号、班级管理、课程管理和平台登录。
- 学生扫码加入、在线抢答、实时全班统计和多人协作。
- 课件直接访问数据库或在文件中保存学生身份。
- 用 HTML Mender 替代完整 LMS、考试系统或教务系统。
- 默认开放任意 JavaScript 编写和任意网络请求。
- 一次性完整复制 PowerPoint 的全部高级能力。

## 假设与非功能要求

- 主要用户是 K12 老师，同时兼容职业教育和教师培训场景。
- 主要运行环境是 Windows 10/11、Chrome、Edge、教室大屏和平板。
- 单个本地项目按最多约 50 页、500MB 资源进行设计和验证。
- 本地服务仅监听回环地址，默认不向局域网开放。
- 编辑过程应自动恢复，保存过程应有版本记录和未保存提醒。
- 互动记录失败不能阻塞课件内容和教学流程。
- 互动配置、事件类型和平台桥接协议必须有版本号。
- 新版本运行时需要兼容已导出的旧课件。
- 所有常用互动需要支持鼠标、触摸和键盘操作。
- 所有动画需要提供减少动画或关闭动画选项。
- 课件文件不得包含学生隐私、平台密钥或数据库凭据。

## 方案比较

### 方案 A：统一互动协议，推荐并已确认

所有互动都由触发、目标、动作、条件、效果、状态和记录类型组成。本地运行时使用本地适配器，未来平台使用平台适配器。

优点：课件无需重做即可接平台；功能可组合；事件可统一记录；长期维护成本较低。

风险：前期需要先建立互动模型、稳定编号和运行时协议。

### 方案 B：每种互动单独写死

每种功能拥有独立按钮、数据和脚本。

优点：少量功能可以较快上线。

风险：功能组合困难；平台接入时需要重复改造；埋点和状态规则容易不一致。

### 方案 C：等待平台完成后再开发互动

先建设账号、数据库和平台接口，再制作课件互动。

优点：平台数据模型可以一次确定。

风险：阻塞本地编辑器发展；老师当前无法制作和验证互动课件。

## 统一互动模型

每个互动由以下字段构成：

- 互动编号：在课件版本之间保持稳定。
- 互动名称：老师可修改，例如“揭晓第二题答案”。
- 所属页面：指向稳定的页面编号。
- 触发元素：文字、图片、按钮、形状、媒体或页面本身。
- 触发方式：点击、双击、悬停、键盘、页面进入、滚动、定时或媒体时间点。
- 目标元素：一个或多个被操作元素，也可以是页面、媒体或弹窗。
- 执行动作：显示、隐藏、切换、跳转、播放、暂停、提交、重置或设置状态。
- 条件：答案、状态、次数、时间、页面路径或其他互动结果。
- 视觉效果：淡入、飞入、缩放、擦除、强调或无动画。
- 初始状态：显示、隐藏、禁用、已完成或自定义值。
- 完成条件：执行一次、达到次数、答对、媒体完成或任务完成。
- 记录类型：不记录、过程记录、结果记录或完整记录。
- 无障碍设置：键盘操作、替代文字、减少动画和朗读标签。

同一元素可以包含多个互动，同一互动也可以同时执行多个动作。

## 教学互动能力库

### 一、基础呈现与课堂讲解

- 点击、双击、悬停、键盘、页面进入、滚动到位置和倒计时结束触发。
- 文字、图片、答案和步骤逐条出现。
- 显示、隐藏、切换显示和再次点击收起。
- 淡入、飞入、缩放、擦除、聚焦、高亮、闪烁和路径移动。
- 标签页、折叠面板、手风琴、轮播图和分步骤面板。
- 卡片翻转、点击揭晓、遮罩揭晓和刮卡式揭晓。
- 图片热点点击后显示说明、媒体、弹窗或局部放大。
- 提示气泡、说明卡片、弹窗、全屏弹窗和侧边抽屉。
- 图片放大、拖动查看、局部缩放、前后对比和透明度对比。
- 聚光灯、遮罩、激光笔和重点区域强调。
- 上一页、下一页、指定页跳转、目录、章节和返回上一处。
- 根据选择进入不同页面或不同讲解分支。
- 自动播放、定时出现、定时跳转、暂停和重新开始。
- 重置当前互动、重置本页或恢复课件初始状态。
- 页面切换后保持互动状态，或由老师指定每次进入重置。

### 二、音频、视频、朗读与互动媒体

- 上传、替换、删除音频或视频，并保持原位置和尺寸。
- 播放、暂停、停止、重播、快进、后退、音量和静音。
- 自动播放、循环、倍速、控制条、全屏和画中画设置。
- 指定音视频播放起止时间。
- 媒体时间点自动暂停并显示问题、提示、图片或说明。
- 视频画面热点和互动视频分支。
- 字幕上传、字幕开关、字幕样式、双语字幕和逐句定位。
- 视频文字稿点击跳转，播放时同步高亮当前句。
- 音频逐句播放、听写、单句循环和间隔播放。
- 课文朗读同步高亮段落、句子或词语。
- 点击元素播放音效、语音讲解、背景音乐或发音示范。
- 原声、教师讲解、伴奏、慢速朗读和多语言音轨切换。
- 图片画廊、360 度图片、前后对比滑块和分层图片讲解。
- SVG 部位点击、变色、高亮、标签和语音说明。
- 媒体加载失败时显示替代图片、文字和重试按钮。
- ZIP 导出自动携带资源并检查缺失文件与格式兼容性。

- 填空、完形填空、短文本、关键词和段落回答。

## 老师的互动设置界面

### 基础流程

1. 老师选中文字、图片、按钮、形状或媒体。
2. 点击工具栏“互动”。
3. 右侧打开互动设置面板。
4. 选择常用模板或自定义互动。
5. 设置触发方式、目标、动作、效果、初始状态和记录类型。
6. 进入学生预览验证效果。
7. 保存当前页版本或整个工作台版本。

### 常用互动模板

- 点击揭晓答案。
- 点击显示或隐藏内容。
- 点击打开说明弹窗。
- 点击跳转指定页面。
- 点击播放音频或视频。
- 内容按顺序逐步出现。
- 视频时间点弹题。
- 答对进入下一步，答错显示提示。

### 元素连线模式

- 设置“点击 A 显示 B”时，先选择触发元素，再选择目标元素。
- 编辑状态显示关系线、方向和互动名称。
- 退出互动设置后关系线自动隐藏。
- 一个触发元素可以连接多个目标和动作。
- 一个目标可以被多个触发条件控制。
- 删除目标前提醒其关联互动，避免产生失效配置。

### 互动清单

- 每页显示互动名称、触发元素、目标、动作和状态。
- 支持重命名、启用、停用、复制、删除和排序。
- 支持按元素、触发方式和记录类型筛选。
- 支持定位到互动对应的页面和元素。
- 支持查看互动顺序和分支路径。
- 常用设置默认展开，高级条件默认折叠。

### 预览与发布检查

- 学生预览不保存测试答案到正式课件状态。
- “重置预览”清除本次测试状态。
- “保存当前状态为初始状态”需要明确确认。
- 发布前检查丢失目标、循环跳转、缺失媒体和无效页面。
- 发布前检查不可键盘操作、缺失标签和高风险动画。
- 发布前检查需要记录但未设置互动编号的项目。

## 状态与保存边界

### 永久课件数据

- 文字、图片、位置、样式和媒体。
- 互动规则、初始状态、动画、条件和记录类型。
- 页面、资源、互动稳定编号和协议版本。
- 保存后进入课件文件和版本历史。

### 临时编辑数据

- 当前选中元素、打开面板和撤销重做栈。
- 未保存内容和崩溃恢复缓存。
- 保存成功后写入正式版本，临时缓存可清理。

### 临时预览数据

- 已揭晓内容、测试答案、当前媒体位置和临时路径。
- 默认在重置、关闭预览或重新加载后清除。
- 只有明确使用“保存当前状态为初始状态”才写入课件。

### 平台学习数据

- 学生作答、正确性、得分、次数、用时和学习路径。
- 平台数据库长期保存并与账号、课程和课件版本关联。
- 断网时平台适配器可以暂存，成功上传后删除待上传缓存。

## 平台事件协议

### 原则

- 埋点是实现方式，协议是课件与平台之间的稳定约定。
- 课件不直接访问数据库，不包含平台密钥。
- 课件只产生标准事件，平台负责补充身份并写入数据库。
- 本地适配器和平台适配器实现同一接口。
- 记录失败不能阻塞课件运行。

### 平台提供的上下文

- 学生账号编号。
- 课程、班级和课堂编号。
- 学习任务和本次尝试编号。
- 课件编号和平台中的发布版本。
- 设备、语言和无障碍偏好。

身份上下文由平台注入，不写入课件包。

### 课件产生的事件

- 页面进入、离开和停留。
- 点击、揭晓、弹窗和页面跳转。
- 答案选择、提交、修改、正确性和得分。
- 提示使用、解析查看、信心和错题重做。
- 媒体播放、暂停、跳转、重复和完成度。
- 拖拽、排序、实验步骤、参数和模拟结果。
- 游戏开始、结束、关卡、积分和完成状态。
- 朗读、录音、作品和教师评价状态。
- 章节完成、课件完成和学习结果。

### 事件公共字段

- 协议版本。
- 唯一事件编号。
- 事件类型。
- 发生时间。
- 课件编号和课件版本。
- 页面编号和互动编号。
- 本次会话和尝试编号。
- 结果数据、状态数据和可选耗时。
- 客户端顺序号和重试次数。

### 最小事件示例

```json
{
  "schemaVersion": "1.0",
  "eventId": "evt-unique-id",
  "type": "answer.submitted",
  "timestamp": "2026-07-11T10:00:00.000Z",
  "coursewareId": "courseware-id",
  "coursewareVersion": "v12",
  "pageId": "page-003",
  "interactionId": "interaction-008",
  "attemptId": "attempt-id-from-platform",
  "sequence": 21,
  "payload": {
    "answer": ["A"],
    "correct": true,
    "score": 1,
    "durationMs": 8400
  }
}
```

学生账号、班级和课程信息由平台桥接层或后端补充，课件事件不自行读取这些数据。

### 平台桥接能力

- 获取当前平台上下文。
- 接收并确认课件事件。
- 保存和恢复学生互动状态。
- 上传作品、录音或图片等受控资源。
- 通知课件平台语言、设备和无障碍偏好变化。
- 通知平台课件完成、页面变化和异常状态。

### 可靠性规则

- 每个事件拥有唯一编号，服务端按编号去重。
- 客户端按顺序号发送，必要时允许批量提交。
- 失败事件进入待上传队列并指数退避重试。
- 上传成功后删除本地待上传副本。
- 切换页面或关闭窗口前尝试刷新关键事件。
- 事件协议升级保持向后兼容。
- 不允许学习记录失败导致页面、媒体或练习无法继续。

## 安全与隐私

- 课件在平台受控容器或沙箱中运行。
- 课件只能通过平台桥接接口提交事件。
- 默认阻止任意外部网络请求和任意脚本注入。
- 学生身份、访问令牌和数据库信息不进入课件文件。
- 平台后端验证身份、课程权限、事件类型和数据范围。
- 录音、图片和作品上传必须由平台授权并展示用途说明。
- 平台负责学习数据访问、导出、更正、删除和保留期限。
- 本地预览不上传学习事件。

## 分阶段开发顺序

### 第一阶段：互动基础与课堂讲解

- 统一互动数据结构和稳定互动编号。
- 本地事件适配器。
- 互动设置面板和每页互动清单。
- 点击显示、隐藏和切换。
- 点击揭晓答案。
- 元素连线模式。
- 页面跳转和返回。
- 弹窗说明、图片和视频。
- 淡入、飞入、缩放和强调动画。
- 学生预览、重置预览和保存初始状态。
- 发布前基础检查。

### 第二阶段：媒体与练习

- 音频完整编辑和播放设置。
- 视频时间点、字幕、文字稿和互动弹题。
- 单选、多选、判断、填空和简答。
- 配对、排序、分类和拖拽归位。
- 提示、反馈、解析、尝试次数和本地评分。
- 条件跳转和基础分支路径。

### 第三阶段：模拟、教师工具与游戏

- 参数滑块、图表和通用模拟组件。
- 数学、科学、地理等学科组件。
- 批注、板书、聚光灯和演讲者工具。
- 计时、随机、翻牌、配对和闯关组件。
- 个人、小组和全班模式配置。

### 第四阶段：平台接入

- 平台桥接适配器。
- 账号、课程、班级和任务上下文注入。
- 事件提交、确认、去重和离线队列。
- 学生状态恢复、作品上传和学习记录。
- 平台沙箱、权限和监控。

### 第五阶段：高级流程

- 动画时间轴和多个元素顺序编排。
- 多条件、多目标和复杂分支流程。
- 多元素框选、组合、对齐和分布。
- 页面新增、复制、删除和重排序。
- 母版、主题和跨页统一样式。
- 复杂虚拟实验和互动流程可视化编辑器。

## 第一阶段验收标准

- 老师不写代码即可完成“点击 A 显示 B”。
- 老师可以清楚看到触发元素、目标元素和互动名称。
- 退出互动设置后页面不显示关系线或干扰框。
- 每个互动可以撤销、重做、复制、删除和保存版本。
- 保存、恢复历史版本和完整 ZIP 导出后互动仍然存在。
- 本地预览不污染正式初始状态。
- HTML 和 ZIP 离线打开时互动正常运行。
- 缺失目标、无效页面和缺失资源能够在发布前发现。
- 鼠标、触摸和键盘均可完成常用互动。
- 动画可以关闭或使用减少动画模式。
- 本地适配器产生包含稳定互动编号的标准事件。
- 平台尚未接入时，记录功能缺失不影响课件运行。

## 决策记录

- 选择 HTML 课件项目作为原生格式，而不是把 HTML 仅作为临时导出格式。
- 选择统一互动协议，而不是每种互动独立写死。
- 选择本地适配器与平台适配器分离，避免未来重做课件。
- 选择课件事件通过平台后端写数据库，不允许课件直连数据库。
- 选择教学设计、编辑缓存、预览状态和学生数据四层分离。
- 选择当前离线编辑优先，暂不开发学生登录和实时课堂网络互动。
- 选择简单互动模板作为默认入口，高级流程默认折叠。
- 选择编辑时显示关系线、授课和预览时隐藏关系线。
- 选择稳定页面编号与互动编号，保证版本和学习记录可追溯。
- 选择记录失败不阻塞教学内容和互动运行。
- 选择无障碍与设备适配作为基础能力，而不是发布后的补充功能。
- 选择分阶段建设，先完成课堂讲解高频互动，再扩展媒体、练习、模拟和平台数据。
