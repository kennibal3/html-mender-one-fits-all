# 页面管理设计

## 已确认目标

- 老师可以在工作台页面管理区和编辑器左侧缩略图栏管理页面。
- 页面可以上下拖动排序，放开后按照新顺序重新编号。
- 触屏或不便拖拽时，可以使用上移、下移按钮完成相同操作。
- 页面列表顶部可以新增标准空白课件页，每一页也可以在当前页后新增空白页。
- 支持复制页面、删除页面和恢复已删除页面。
- 页面操作不能破坏已有版本、互动配置、项目资源和 ZIP 导出。

## 数据规则

- `page.id` 是稳定身份，不因排序或显示编号变化。
- `project.pages` 按当前教学顺序保存正常页面。
- `project.deletedPages` 保存进入回收站的页面、原位置和删除时间。
- 页面显示名称根据当前位置生成“第 N 页”，文件名和稳定 ID 不随编号变化。
- 新增空白页创建新的 HTML 文件、可编辑文件和独立 `v001` 初始版本。
- 复制页面复制干净 HTML 与页面互动，但使用新的页面 ID、文件名和版本历史。
- 删除页面只从正常页面列表移入回收站，不删除源码、资源或历史版本。
- 项目必须至少保留一个正常页面。

## 操作与接口

- 新增：在指定页面后插入标准 1920×1080 空白课件页。
- 复制：在当前页面后插入副本。
- 排序：一次提交完整稳定页面 ID 顺序，服务端验证无缺失、重复或未知 ID。
- 删除：检测页面跳转互动引用；存在引用时先返回影响信息，再由用户二次确认。
- 恢复：恢复到删除前位置；原位置不可用时追加到末尾。
- 所有操作完成后重新生成页面导航、缩略图信息和可编辑页面。
- 元数据继续使用临时文件原子替换，避免应用中断造成索引损坏。

## 界面规则

- 工作台页面卡片提供拖动手柄、上移、下移、在此页后新增、复制和移入回收站。
- 页面列表顶部提供“新建空白页”和“回收站”。
- 拖动时显示明确插入位置，保存中禁用重复操作，成功后显示反馈。
- 编辑器左侧缩略图栏提供相同的新增、复制、删除和排序入口。
- 有未保存编辑时，页面管理操作先触发现有保存提醒。
- 回收站页面不参与页码、前后页导航和 ZIP 导出。

## 非功能要求

- 支持最多约 50 页、500MB 的本地项目。
- 页面管理只监听本地回环服务，不增加云端传输。
- Windows 10/11、Chrome、Edge、鼠标、触摸和键盘均可操作。
- 失败操作不允许留下半写入文件或不完整页面索引。
- 页面操作需要自动化测试，并在真实浏览器中验证拖拽与移动按钮两条路径。

## 决策记录

- 选择服务端权威页面模型，而不是只在浏览器内修改页面顺序，保证重启后可靠恢复。
- 选择稳定页面 ID 与动态显示编号分离，避免排序破坏版本和互动关系。
- 选择非破坏性回收站，而不是直接删除页面文件，降低老师误操作风险。
- 新建页面采用标准空白页，不尝试从任意 HTML 中自动识别并清空主要内容。
- 同时提供拖拽和移动按钮，兼顾 PPT 操作习惯、触屏和无障碍使用。
- 页面管理同时出现在工作台和编辑器侧栏，两处调用同一套接口。
