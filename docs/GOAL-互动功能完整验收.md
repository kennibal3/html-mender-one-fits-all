## 0. 项目背景(只读,帮助你理解上下文)

HTML Mender 是面向教师的**本地** HTML 课件编辑器:导入 HTML/ZIP → 多页管理(缩略图、排序、版本历史)→ 元素级编辑 → **互动编辑** → 导出标准 HTML/ZIP →(可选)Electron Windows 打包。当前为单机 + 本地文件存储。

**本轮只做一件事:把"互动功能"做到可交付质量。** 不碰账户、云端、埋点、Windows 发布等其它方向。

### ⚠️ 本项目最容易掉的坑(先读,这决定成败)

编辑时的预览和导出后的课件是**两套代码路径**:

- `vendor/html-slide-mender/assets/html-slide-mender-runtime.js` —— 编辑时的可视化预览。
- `vendor/html-slide-mender/assets/html-slide-mender-interactions.js` —— **导出课件后**在浏览器里实际运行互动的逻辑。

> 历史上"功能做不全 / 导出后失灵"几乎都来自**只改了 runtime、没同步改 interactions**。
> **硬性要求:任何互动的逻辑,必须在这两条路径上行为一致,并且各自有测试。**

相关文件参考:`src/server.js`(服务/上传/项目/版本/导出接口)、`src/core.js`(项目生成/页面管理/编辑页注入)、`public/`(前端工作台)、`test/`(自动化测试)。

## 1. 核心设计理念:两层互动

互动分两层。**必须理解这个分层,否则做出来的东西和 PPT 没区别。**

### A 类:PPT 平替(基础盘)

这 5 种 PPT 基本都能做,**不是产品卖点,但目前有 bug、导出后会失灵,必须先补全到扎实**。

| 编号 | 互动 | 说明 |
|---|---|---|
| A1 | 点击显示/隐藏 | 点触发元素,目标元素显示或隐藏 |
| A2 | 弹窗 | 点触发元素,弹出内容层 |
| A3 | 页面跳转 | 跳到课件内指定页 |
| A4 | 网址跳转 | 打开外部 URL |
| A5 | 逐步讲解 | 元素按预设顺序逐步显示 |

### B 类:HTML 独有(产品护城河,🌟 专门设计并在代码/UI 中标注)

这些是 **PPT 做不到**的能力,是本产品区别于传统课件的根本理由。本轮正式纳入 **B1–B8**，并与复杂场景架构交错实施。

| 编号 | 互动 | 能力 | PPT 做不到的点 |
|---|---|---|---|
| **B1** 🌟 | 答题判定 + 即时反馈 | 学生选/填答案,页面立刻判对错并给解析(单选/多选/填空/判断) | PPT 不能"判断"输入,只能按预设顺序放动画 |
| **B2** 🌟 | 条件分支 | 按答案对错 / 选择 / 变量,走不同页面或显示不同内容 | PPT 超链接是死的,不能按对错决定去向 |
| **B3** 🌟 | 拖拽交互 | 运行时拖动匹配、分类、排序、拖入空格 | PPT 只有做课件时能拖,学生播放时拖不动 |
| **B4** 🌟 | 输入即时响应 | 文本、数字、选项或滑块变化后实时计算并更新页面 | PPT 不能根据连续输入实时计算和反馈 |
| **B5** 🌟 | 状态与计分 | 按学习者保存变量、分数、进度、尝试和解锁状态 | PPT 不能可靠保存个人学习状态和跨场景进度 |
| **B6** 🌟 | 随机化 | 抽题、题序、选项和互动项目按规则随机 | PPT 难以为每次练习生成可恢复的稳定随机内容 |
| **B7** 🌟 | 计时与倒计时 | 支持限时、用时、暂停规则、超时反馈和恢复 | PPT 缺少可保存、可分支的学习任务计时 |
| **B8** 🌟 | 媒体精确联动 | 按媒体时间点或时间段弹题、显隐、暂停和跳转 | PPT 无法可靠组合媒体时间轴与复杂学习状态 |

> **标注要求:** B 类互动在配置界面、导出 HTML 的 data 属性、以及代码注释中,都要有清晰可识别的标记(例如 `data-hm-interaction="quiz"` 并在 UI 上以区别于 A 类的样式呈现),让"这是 HTML 独有能力"在产品里显性可见。

### B 类与复杂场景的共同边界

- B1–B8 必须可以运行在普通页面、活动节点弹窗和已支持的本地/同源场景中。
- B2 的目标不只限于平面页面，还可以是弹窗、iframe 或动态场景。
- B5 使用项目—页面—场景—测试四层状态，并通过统一存储接口保存。
- 本地阶段支持同一电脑上的多个学习者姓名或编号档案；未来云端账户使用同一状态接口扩展，不在本轮建设云服务。
- 导出课件支持课堂演示模式和学生练习模式。演示模式不记录个人成绩，练习模式按本地学习者隔离保存。
- 跨域 iframe、封闭组件、加密脚本和 Canvas 内部对象只管理外层，不承诺深入编辑。

## 2. 六阶段生命周期验收模型(适用于每一种互动)

**每一种互动都必须走完这 6 个阶段,每个阶段 = 一格,每格都要有"通过证据"才算完成。**

| 阶段 | 名称 | 该格要验证的内容 |
|---|---|---|
| ① | 设置 | 能在编辑器里正确配置该互动;非法配置被拦截并给出明确提示(例:同一元素不能既是点击触发器又是被隐藏的逐步显示步骤) |
| ② | 测试预览 | 配置后进入真实测试态:隐藏编辑框/复杂面板、锁定文字图片位置编辑、顶部显示"测试互动 / 返回互动编辑"工具条;跳页与网址跳转在测试时**先显示目标、不强制离开未保存页面**;返回后恢复原状态 |
| ③ | 保存 | 互动配置随**页面版本**持久化,保存后不丢失 |
| ④ | 重开 | 关闭 app,从最近任务重新打开,互动配置与行为完好可用 |
| ⑤ | 版本恢复 | 恢复某历史版本时,**互动配置一并回滚到该版本对应的状态**(已确认:恢复要跟着回滚) |
| ⑥ | 导出后独立运行 | 导出 HTML/ZIP → **浏览器双击直接打开** → 互动正常。**完全脱离 `npm start` 本地服务,双击即用**;控制台 **0 报错**(已确认:必须可完全双击使用) |

### B 类额外第 ⑦ 格:逻辑正确性(A 类不要求,B 类必须)

| ⑦ | 逻辑正确性 | 判定/分支/命中的结果在**所有输入**下都正确——覆盖正例、反例、边界、非法输入。**杜绝"点了有反应但答案永远判对"这类 runs-but-wrong。** |

## 3. 逐项规格与验收标准

> 通用规则:下方每一项都要跑完第 2 节的 6 格(B 类 7 格)。凡与通用定义相同的部分不再重复,只写**该互动特有的配置、行为、边界与断言**。

### A1 点击显示/隐藏
- **配置:** 选触发元素 + 目标元素 + 动作(显示/隐藏/切换)。
- **特有验收:** ①拦截"触发器==目标"这类自指非法配置;⑥导出后点击触发,目标正确显隐,且初始隐藏元素在文件打开时就是隐藏的(不能先闪现)。

### A2 弹窗
- **配置:** 触发元素 + 弹窗内容(可含文字/图片)+ 关闭方式。
- **特有验收:** 弹窗层级正确(不被其它元素遮挡);⑥导出后可打开可关闭;移动端/触摸也可关闭。

### A3 页面跳转
- **配置:** 触发元素 + 目标页。
- **特有验收:** ②测试态先显示目标页、不强制离开未保存页;⑥导出后跳转到正确页;目标页被删除后给出降级处理(不报错崩溃)。

### A4 网址跳转
- **配置:** 触发元素 + URL。
- **特有验收:** ②测试态先显示目标 URL、不直接跳走;⑥导出后正常打开;非法/空 URL 有防御。

### A5 逐步讲解
- **配置:** 一组元素 + 显示顺序。
- **特有验收:** ①拦截"同一元素既作点击触发器又作逐步步骤";按顺序逐步显示;⑥导出后逐步逻辑完整、可从头走到尾。

---

### B1 🌟 答题判定 + 即时反馈

- **题型:** 单选、多选、填空、判断。
- **数据模型(建议):** `{ type, question, options[], correctAnswer(s), matchRule, explanation }`;填空 `matchRule` 支持:精确 / 忽略大小写 / 忽略首尾空格 / 多个可接受答案。
- **运行行为:** 学生作答 → 立即判对错 → 显示对/错样式 + 解析。两条运行时路径行为一致。
- **② 测试预览:** 保存前即可在编辑器内真实作答验证。
- **⑦ 逻辑正确性断言(必须全部覆盖):**
  - 单选:选正确项→判对;选错误项→判错。
  - 多选:**全部选对且不多选**→对;漏选→错;多选→错。
  - 填空:接受答案→对;非接受→错;边界:空输入、前后空格、大小写按 `matchRule` 生效。
  - 判断:对/错各验一次。
  - **反例保证:** 构造错误答案确认**不会被误判为对**。

### B2 🌟 条件分支

- **依赖:** 通常基于 B1 的判定结果,故排在 B1 之后。
- **触发源:** B1 答题结果(对/错)/ 用户选择 / 变量值。
- **数据模型(建议):** `{ condition, then: { action: "gotoPage" | "showBlock", target } }`。
- **运行行为:** 满足条件走 A 路径(如答对进阶),否则走 B 路径(如答错补救)。
- **⑦ 逻辑正确性断言:**
  - 答对 → 去到"进阶"目标;答错 → 去到"补救"目标(各验一次)。
  - 多分支时命中唯一正确分支。
  - **边界:** 无任何分支命中时,有明确定义的默认行为(不得静默失败或崩溃)。

### B3 🌟 拖拽交互

- **模式:** 配对 / 分类 / 排序 / 拖入空格。
- **数据模型(建议):** `draggables[]` + `dropTargets[]` + `correctMapping`。
- **运行行为:** 学生运行时可拖动(**鼠标 + 触摸都要支持**),命中判定,吸附或退回;可给出对错反馈。
- **⑦ 逻辑正确性断言:**
  - 正确放置 → 判对;错误放置 → 判错或退回。
  - **部分正确**有明确处理(如逐项判定)。
  - **边界:** 拖放目标重叠、快速拖动、触摸设备下命中判定准确。

### B4 🌟 输入即时响应

- **输入类型:** 文本、数字、单选/多选、下拉选项和滑块。
- **规则方式:** 只允许教师通过可视化规则配置变量、公式和页面响应，不开放任意 JavaScript。
- **安全公式:** 支持加减乘除、比较、区间、四舍五入、最大值、最小值及经明确列入白名单的函数。
- **运行行为:** 输入变化 → 校验输入 → 重新计算 → 更新文字、样式、显隐、进度或允许的场景动作。
- **⑦ 逻辑正确性断言:** 正常值、空值、非法数字、边界值、超出区间和多个输入联动结果均正确；非法公式在配置阶段被拦截，导出后不执行用户脚本。

### B5 🌟 状态与计分

- **状态模型:** 项目、页面、场景、测试四层状态；同名元素不自动共享变量。
- **计分能力:** 题目分值、部分得分、尝试次数扣分、总分、进度和达标解锁。
- **本地身份:** 同一课件支持多个本地学习者姓名或编号档案，答案、分数、计时和进度彼此隔离。
- **启动模式:** 课堂演示模式不要求身份且不记录个人成绩；学生练习模式选择或新建档案并记住上次选择。
- **持久化:** 通过带版本号的统一存储接口写入当前浏览器数据库；保留未来替换云端账户存储的适配边界。
- **⑦ 逻辑正确性断言:** 满分、部分得分、扣分关闭/开启、最低分限制、重复作答、解锁阈值、档案切换、重开恢复和清除进度均符合配置；教师测试状态不得污染学生成绩。

### B6 🌟 随机化

- **范围:** 从题库抽题、题目顺序、选项顺序、分类/配对/拖拽项目顺序。
- **稳定规则:** 同一学习者同一次练习使用稳定随机种子，刷新、重开和恢复进度后结果不变化；选择重新练习后才生成新结果。
- **统一讲解:** 教师可以临时锁定固定顺序，教师预览不消耗或改变学生随机结果。
- **⑦ 逻辑正确性断言:** 数量边界、空题库、抽取不重复、固定种子可复现、不同场次可变化、档案之间互不污染，并保证随机化后正确答案映射仍然准确。

### B7 🌟 计时与倒计时

- **模式:** 正计时、倒计时和总用时。
- **离开规则:** 每个计时器分别配置离开页面、弹窗或标签页后暂停或继续。
- **超时动作:** 通过可视化规则执行判错、交卷、显示提示或进入指定场景。
- **恢复:** 学生练习模式按学习者和场次保存剩余时间；演示预览计时不写入正式记录。
- **防御:** 检测系统时间明显变化、重复启动、多个计时器和场景销毁，避免静默产生错误结果。
- **⑦ 逻辑正确性断言:** 暂停/继续两种规则、刷新恢复、切换档案、到零仅触发一次、超时分支和系统时间异常均有确定行为。

### B8 🌟 媒体精确联动

- **互动时间轴:** 教师在音视频时间点或时间段上配置弹题、显示隐藏、暂停、跳转和场景进入。
- **轻量编辑:** 支持开头结尾裁剪、分段、排序、音量、淡入淡出、封面、字幕和简单画面裁切。
- **范围边界:** 不建设专业多轨、绿幕、复杂特效和专业关键帧；允许导入专业工具制作的成品。跨域或 DRM 媒体只使用浏览器允许的外层控制。
- **运行行为:** 拖动进度、重复播放、倍速、暂停后恢复和跨场景返回时，时间点事件不得无意重复或遗漏。
- **⑦ 逻辑正确性断言:** 时间点前后边界、跨越触发点、回退重播、片段结束、字幕同步、多个事件同一时刻和导出媒体路径均正确。

## 4. 已知的验收样例(作为写测试的模板)

**样例一(A1 × ⑥ 导出后独立运行):**
> 前置:一页,元素 A=触发器,元素 B 初始隐藏,配"点 A 显示 B",已保存。
> 操作:导出完整 HTML → 浏览器**双击**打开 → 点 A。
> 期望:B 从隐藏变为显示;打开瞬间 B 不闪现;控制台无报错;全程不依赖本地服务。
> 证据:自动化测试打开导出文件、模拟点击、断言 B 可见;并人工按此脚本手点一遍。

**样例二(B1 × ⑦ 逻辑正确性,多选题):**
> 前置:多选题,正确答案 = {选项1, 选项3}。
> 用例:选{1,3}→对;选{1}→错(漏选);选{1,3,4}→错(多选);选{2,4}→错。
> 证据:自动化测试逐一断言四种组合的判定结果。

**样例三(全链路闭环,发布前必须人工走一遍):**
> 设置 B1 答题 → 保存 → 关闭 app → 重开(确认还在)→ 恢复一个更早的历史版本(确认互动跟着回滚)→ 导出 ZIP → 解压后双击 HTML → 作答确认判定正确、无报错。

## 5. Codex 执行硬规矩(专治"做不全 / 有 bug")

1. **深度优先,禁止广度优先。** 除已经确认的场景、状态、坐标、随机、调度和媒体公共基础外，一次只完成**一种**互动，把它的 6 格(B 类 7 格)全部跑通并留证据，才允许开下一种。**禁止多种互动一起做到一半。**
2. **每格都要有证据。** 证据 = 一条自动化测试 **或** 一段可复现的人工验证脚本。**没有证据的格子一律不算完成**,不许口头声称"应该没问题"。
3. **两条运行时路径都要改到。** 任何互动逻辑,`runtime.js`(编辑预览)与 `interactions.js`(导出运行)都要实现且行为一致,各有测试。改一条、漏一条,视为未完成。
4. **不许越界。** 不改与本任务无关的代码；不加任何未在本文件要求的功能；不建设云端账户、专业多轨剪辑或跨域内容内部编辑。
5. **不许破坏用户数据。** 任何保存/导出/恢复操作,都不得损坏用户已有课件;测试过程不得写坏真实数据。
6. **规格不清就提问。** 遇到本文件没定义的行为,停下来问,不要自行假设后一路做偏。

## 6. 执行顺序与"完成的定义"

### 执行顺序(场景架构与 B 类互动交错)

1. **最小场景模型 + 活动节点弹窗**：打通“首页 → 弹窗 → 内部互动 → 返回”。
2. **B5 状态基础设施**：先建设四层变量接口、本地存储适配器、多个学习者档案及演示/练习模式。
3. **B1 + B5 计分闭环**：完成答题判定后补全计分、进度和解锁。
4. **跨场景导航 + B2**：加入多 HTML、同源 iframe 和场景目标，再完成条件分支。
