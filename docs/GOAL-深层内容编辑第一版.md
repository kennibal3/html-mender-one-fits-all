# GOAL：HTML Mender 深层内容编辑（第一版）

> 本文件是本切片的**唯一规格**。逐项执行、逐格汇报，不许跳格，不许自行扩大范围。
> 规格没写清楚的地方，先停下来问使用者，不要自行假设。
> 本版踩在 A1–A5（Goal A）之上：动手前 A1–A5 必须已在平面模型里稳定并持续通过回归。

## 1. 本版定位

真实老师课件常有“弹窗”和“跳转到另一个本地 HTML 子页面”。当前“页面 + 顶层元素”模型进不去这些深层内容，典型表现是**首页互动正常、但弹窗或子页面里的互动失效**。

本版只解决一个高频痛点：**让老师能进入弹窗内部和同源子页面，在里面编辑元素、配置 A1–A5 互动，并且导出后仍能独立跑通。** 其余深层类型（iframe 深嵌、SPA、Shadow DOM、Canvas 等）一律不在本版，留待真实需求出现再逐条拉入。

设计遵循来源文档《HTML Mender 复杂场景编辑设计》的已确认决策，但只取本版必需的最小子集。

## 2. 目标用户：三档 HTML 理解程度

产品默认体验必须按“零基础”设计，进阶能力藏起来、不打扰。

| 档位 | 老师是谁 | 本版必须给到的体验 |
| --- | --- | --- |
| T1 零基础（大多数） | 完全不懂 HTML、路由、DOM | 全程无技术术语；进入深层用“点进去继续编辑”等大白话；面包屑显示课件里的**可见标题**，不显示文件名或技术 ID；诊断完全隐藏；进不去的内容只给一句人话解释 |
| T2 懂一点 | 知道“页面”“链接”“弹出框” | 能看懂左侧场景树是“课件里一层层的画面”；能手动框选某块“作为独立场景编辑” |
| T3 进阶 | 做过简单 HTML、想调细节 | 可在任务设置里开“高级维护模式”，才显示诊断、路径重放、进不去的技术原因 |

**验收硬性要求：** T1 的完整操作路径（打开→点进弹窗→改内容/配互动→返回→导出）全程不得出现任何技术术语或技术 ID。

## 3. 本版范围

### 3.1 做（IN）

- 进入并编辑**弹窗内部**的元素与 A1–A5 互动。
- 进入并编辑**同源子 HTML 页面**的元素与 A1–A5 互动。
- 两种进入方式：像真课件一样**直接点击进入**，以及从**左侧场景树**进入（入口损坏时也能强制进入）。
- **面包屑**显示当前位置（用可见标题）。
- 返回父场景时恢复滚动位置、弹窗层级、焦点与测试状态。
- **活动节点弹窗**：弹窗默认显示真实活动节点，不克隆内部内容。
- 自动草稿 + 手动正式版本，含场景关系与互动配置；版本恢复时互动/场景一并回滚。
- 导出 HTML/ZIP：**深层内容里的互动也要独立跑通**，浏览器双击即用、不依赖本地服务。
- 诊断默认对老师隐藏，可选高级维护模式。
- 无法安全深入的内容只管理外层，并给一句可理解的限制说明。

### 3.2 不做（OUT）

- 同源 iframe 内部编辑（紧随其后的候选，由样本库决定是否提前）。
- SPA/Hash/History 路由视图、iframe 深层嵌套、Shadow DOM、Canvas/小游戏、框架响应式重渲染内容。
- B1–B8 全部互动（B1 为推荐的下一切片）。
- 多学习者档案、演示/练习模式、可信项目模式、轻量媒体编辑、A 档大规模性能优化。

> 边界原则：**框架响应式渲染出来的内容不在本版。** 直接改 DOM 会被重渲染冲掉；本版只处理静态或点击后注入的普通 DOM。

## 4. 最小场景模型

```text
项目
└── 页面（本地 HTML）
    └── 弹窗 或 同源子页面
        └── 内部元素与 A1–A5 互动
```

每个场景至少记录：稳定场景 ID、显示用的可见名称、类型（弹窗/子页面）、所属 HTML 文件、父子关系、进入动作或条件、对应元素选择器/节点标识、编辑补丁与互动配置、进入退出规则、来源与可信度说明。

多个弹窗可引用同一真实 HTML 文件；场景不是文件副本。

## 5. 场景发现（本版从简）

- **静态扫描：** 扫本地 HTML、同源链接、常见隐藏弹窗结构，建初始场景树，记可信度。
- **运行发现：** 仅监测当前活动页面里“点击后出现的弹窗/子页面”，记录进入路径；新场景先标“待确认”，不直接污染正式结构。
- **人工补充（可靠主路径）：** 老师可框选区域“作为独立场景编辑”，可合并误识别、可标记为普通动画变化。
- 本版不做跨任意 HTML 的全自动枚举、复杂去重和循环检测。

## 6. 保存、版本与导出

- 保存：自动草稿（实时增量，切换场景不丢）+ 手动正式版本（可命名、可恢复）。一个真实 HTML 只存一份；编辑补丁精确作用于对应文件与元素；保留原始导入文件作为可恢复基线。
- 恢复历史版本前自动建临时恢复点；版本恢复结构与配置（含互动/场景），不恢复某次临时测试状态。
- 导出：确认后的修改写入导出副本；本地互动运行逻辑 + 场景运行清单随 ZIP 导出；导出课件不依赖 HTML Mender 或任何服务即可运行；导出页面不含编辑器工具条。
- 导出前检查：页面/子页面可达性、互动目标存在、资源路径、文件名大小写、深层内容运行一致性；只有阻断性问题才打扰老师。

## 7. 测试样本库

**必须先建一个真实代表性课件样本库，所有验收都对着它跑，而不是对着“任意 HTML”。**

- 收集 5–8 个真实老师课件，覆盖：含弹窗、含子页面跳转、点击后动态出现内容，以及一个含无法深入内容（如跨域 iframe）的反例。
- 样本库同时回答：同源 iframe 是否提前进入本版。如果样本里普遍存在同源 iframe，再决定拉入；没有则维持 backlog。
- 原课件保持只读；自动化测试使用脱敏后的最小复现样本。

## 8. 验收标准

### 8.1 每类深层场景都要走通的闭环

弹窗、同源子页面两类场景分别满足：

1. 导入后能被发现，或老师能人工登记。
2. 能从场景树和真实点击两条路径进入。
3. 能在深层场景里选中并编辑元素（文字、图片等）。
4. 能在深层场景里配置、预览、返回 A1–A5 互动。
5. 返回父场景后位置和状态正确。
6. 自动草稿和正式版本均可恢复。
7. 关闭应用后重新打开，场景关系与互动不丢。
8. HTML/ZIP 导出后，深层内容里的互动无需编辑器即可独立运行、console 零报错。
9. 原课件脚本未被无提示删除或覆盖。
10. 无法深入的内容有明确、可理解的限制说明。

### 8.2 强制回归路径

- 首页 → 弹窗 → 弹窗内按钮 → 显示隐藏（A1）→ 返回首页，状态恢复。
- 首页 → 同源子 HTML → 子页面内互动 → 返回上一页，状态恢复。
- 保存草稿 → 保存正式版本 → 关闭重开 → ZIP 导出 → 独立运行，深层互动仍有效。
- 原脚本兼容：含原互动的课件导入后，原互动未被破坏。
- A1–A5 全部平面模型回归仍通过。

### 8.3 友好性验收（T1 视角）

- T1 完整路径全程无技术术语、无技术 ID。
- 面包屑用可见标题。
- 诊断默认不可见；仅开高级维护模式后出现。

## 9. 两条运行时路径

深层内容的互动逻辑必须在两条路径上同时实现、行为一致、各有测试：

- `vendor/html-slide-mender/assets/html-slide-mender-runtime.js`：编辑时预览。
- `vendor/html-slide-mender/assets/html-slide-mender-interactions.js`：导出课件后实际运行。

深层内容若只在预览实现、没有进入导出运行路径，即算未完成。

## 10. Codex 硬规矩

1. **深度优先。** 先把“弹窗内部”十步闭环全部跑通留证据，再做“同源子页面”；禁止两类同时做到一半。
2. **每格有证据。** 证据为对着样本库通过的自动化测试或可复现端到端脚本。
3. **两条运行时路径都要改到。**
4. **导出后深层内容可双击独立运行，console 零报错。**
5. **不许破坏原课件。** 不无提示删除或覆盖原脚本、内联事件；保存、导出、恢复不得损坏用户课件。
6. **不许越界。** 不做 §3.2 的任何内容，不加未要求的功能。
7. **动手前先读现有页面管理、版本、导出、互动实现并对齐。**
8. **规格不清就停下来问。**

## 11. 执行顺序与完成定义

### 顺序

1. 前置确认：A1–A5 已稳定、回归通过。
2. 建 §7 真实样本库。
3. 建最小场景模型 + 活动节点弹窗，打通“首页 → 弹窗 → 内部 A1–A5 → 返回”整条闭环。
4. 扩展到同源子页面，打通对应闭环。
5. 补齐场景树、面包屑、人工补充和高级维护模式隐藏，完成友好性验收。
6. 统一加固自动草稿、正式版本、独立 ZIP 和导出前检查，跑全量测试与安全依赖检查。

> 顺序解释：§8.1 要求弹窗同时支持“真实点击”和“场景树”进入。因此弹窗阶段允许实现满足闭环所必需的**最小场景树与可见标题面包屑**；第 5 步再补齐人工维护和完整友好性验收，不把子页面提前做成半成品。

### 最小场景树实施决策（2026-07-20 已确认）

- 直接把现有“页面列表”升级为“课件画面”，不新增第二套场景面板；页面缩略图、排序、复制和删除继续保留。
- 当前页面下按父子关系展示弹窗节点；教师可从树中强制进入，嵌套弹窗按祖先到子级的顺序进入。
- 场景树和面包屑只显示课件中的可见标题，例如“首页 › 课程介绍”；不得向 T1 显示文件名、选择器、场景 ID、可信度或诊断信息。
- 进入弹窗时移动真实节点而非克隆内容，并维护独立进入栈；逐层返回时把节点放回原位置，保留原事件、表单和播放器状态。
- 场景打开期间保存、复制或导出不得遗漏、复制或损坏真实节点；目标缺失、重复或已被移除时保持当前画面并显示一句人话说明。
- 本格只实现编辑器内的场景导航，不改变导出互动行为；深层互动开始执行时再同时修改并测试 §9 的两条运行时路径。
- 本格不含弹窗内部元素编辑、A1–A5 深层配置、人工登记、高级诊断、子页面场景，以及完整滚动/焦点/互动测试状态恢复。

**非功能约束：** 单项目按数十个场景设计，进入时按需在当前文档查找；不执行场景字符串代码、不访问外部内容；所有交互支持键盘操作，失败不改变课件 DOM，编辑器导航界面不得进入导出文件。
